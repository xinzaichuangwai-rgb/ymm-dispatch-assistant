☁️ 云端编译APK教程

全程在网页上操作 · 不用装Java · 不用装Android Studio · 不用装任何软件

📖原理说明

你的电脑里没有Java和安卓开发工具，编译不了APK。但 GitHub（全球最大的代码托管平台）提供免费的云端编译服务，我们只要把源码传上去，它会在云端的电脑里帮我们编译好APK，然后我们下载就行。

上传源码到GitHub → 云端自动编译 → 下载APK → 装到手机

整个过程类似"把U盘里的文件交给打印店，打印店帮你打印好你去取"。

1注册GitHub账号

  1. 电脑浏览器打开 github.com
  2. 点右上角 Sign up（注册）
  3. 填邮箱、设密码、取用户名
  4. 验证邮箱（去你邮箱点确认链接）
💡 已有GitHub账号的直接登录，跳到第二步。
📍 github.com 首页 → 右上角 Sign up

2创建新仓库（上传代码的地方）

  1. 登录后，点右上角头像旁的 「＋」 → 选 New repository
  2. Repository name 填：ymm-dispatch-assistant
  3. 下面选 Public（公开，否则不能免费编译）
  4. 其他不用改，点最下方绿色的 Create repository 按钮
📍 创建仓库页面：填名字 → 选Public → Create

3上传源码文件

这一步最关键。把AndroidApp文件夹里的所有文件上传到GitHub，保持目录结构不变。

操作方法：
1. 在刚创建的仓库页面，点 「uploading an existing file」 链接
2. 打开电脑上的 AndroidApp 文件夹
3. 把里面所有文件和文件夹拖到网页上传区域
4. 注意：.github 文件夹也要传上去（里面是编译脚本）
5. 等待上传完成（文件不多，几秒钟）
6. 拉到最下面，点绿色的 Commit changes 按钮
⚠️ 重要：必须保持以下目录结构：
ymm-dispatch-assistant/
├── .github/workflows/build-apk.yml  ← 编译脚本（最重要！）
├── app/
│   ├── build.gradle
│   └── src/main/
│       ├── AndroidManifest.xml
│       ├── java/com/ymm/dispatch/
│       │   ├── MainActivity.java
│       │   └── FloatingWindowService.java
│       └── res/
│           ├── layout/
│           │   ├── activity_main.xml
│           │   └── floating_window.xml
│           ├── values/
│           │   ├── strings.xml
│           │   └── styles.xml
│           ├── drawable/
│           │   ├── ic_launcher_background.xml
│           │   └── ic_launcher_foreground.xml
│           └── mipmap-hdpi/
│               └── ic_launcher.xml
├── build.gradle
├── settings.gradle
├── gradle/wrapper/gradle-wrapper.properties
└── local.properties（自动生成，不用传）
💡 GitHub网页上传小技巧：
• 可以一次拖多个文件
• 文件夹要一个一个传，先建好对应目录
• 如果网页上传太麻烦，可以用 GitHub Desktop 软件（免费），拖拽更方便

4等待云端编译

  1. 文件上传完成后，点仓库页面顶部的 「Actions」 标签
  2. 左边会看到 Build APK，点它
  3. 右边会看到一个正在运行的任务（黄色圆圈🔄）
  4. 等待 3-8分钟（第一次编译要下载Android SDK，会慢一点）
  5. 看到 绿色对勾 ✅ 就说明编译成功了
📍 Actions标签 → Build APK → 等待绿勾✅
⚠️ 如果编译失败（红叉❌）：
• 点进去看报错信息
• 把报错截图发给我，我帮你修
• 常见原因：文件没传全、目录结构不对

5下载APK

  1. 点进编译成功的那个任务（绿勾的）
  2. 拉到页面最下方，找到 「Artifacts」 区域
  3. 点击 ymm-dispatch-assistant-apk
  4. 浏览器会下载一个 .zip 文件
  5. 解压这个zip，里面就是 app-debug.apk
💡 这就是你的APK安装包了！可以发到手机上安装使用。

6安装到手机

  1. 把 app-debug.apk 文件发到手机上（微信传 / QQ传 / 数据线拷贝）
  2. 在手机上点击这个 apk 文件
  3. 手机会提示"为了安装，请允许安装未知来源应用"→ 允许
  4. 点 安装 → 等待安装完成
  5. 桌面上会出现 「抢单助手」 图标

7首次使用

  1. 打开「抢单助手」
  2. 点 「🚀 启动悬浮窗」
  3. 弹窗问是否允许截屏 → 允许（每次启动都要授权一次）
  4. 弹窗问是否允许录音 → 允许
  5. 弹窗问是否允许悬浮窗 → 去设置里开启 "显示在其他应用上层"
  6. 回到App，再点一次「启动悬浮窗」
  7. 打开运满满 → 悬浮窗出现在最上层
💡 悬浮窗操作：
• 按住标题栏可以拖动位置
• 点 「─」 折叠成一条（不挡视线）
• 点 「📷抓取」 截屏识别运满满页面
• 点 「🎤听音」 监听运满满语音播报
• 点 「📲运满满」 秒切回运满满App

❓常见问题

Q：GitHub注册页面是英文的，看不懂？
A：用Chrome浏览器打开，右键选"翻译成中文"。或者记住几个关键词：Sign up=注册，New repository=新建仓库，Actions=编译记录，Artifacts=下载文件。
Q：上传文件太麻烦了，文件夹传不上去？
A：GitHub网页版确实不太好传文件夹。两个办法：
① 装 GitHub Desktop（免费软件），拖拽上传更方便
② 把我帮你做的整个 AndroidApp 文件夹打包成zip，我可以帮你用命令行推送到GitHub
Q：编译失败了怎么办？
A：点进Actions里失败的任务，看红色报错信息，截图发给我。我看了报错就能帮你改代码。
Q：编译成功但APK装不上？
A：手机设置 → 安全 → 允许"安装未知来源应用"。华为/小米手机可能需要在"纯净模式"里关掉限制。

📋一句话总结

注册GitHub → 创建仓库 → 上传AndroidApp文件夹 → 点Actions等编译 → 下载APK → 装手机

整个过程大约15-20分钟（主要是等编译和上传文件），全程不用装任何开发软件。
